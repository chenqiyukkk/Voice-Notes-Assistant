Workflow

从音频到纪要，保持一条可验证链路

笔记中自动插入

```lecture-audio
file: recordings/recording-2026-02-26-101530-233.wav
title: 计算机组成原理
duration: 00:47:12
```

转写缓存：.transcript.json
纪要缓存：.summary.md

  1. 01

    开始录音

    命令 开始/停止录音 或点击 Ribbon。支持暂停/继续，默认保存到 recordings/。

  2. 02

    自动嵌入播放器

    录音结束后占位块会自动替换为可播放的 lecture-audio 音频块。

  3. 03

    执行转写

    支持 OpenAI Whisper、iFLYTEK 和本地 whisper.cpp，命中缓存时会直接复用结果。

  4. 04

    生成纪要

    支持 OpenAI-compatible 与 Claude，输出结构化 Markdown，支持分段合并长文总结。

  5. 05

    时间戳复习

    命令 插入时间戳，点击 callout 即可跳转到当前活跃播放器的对应时间。

  6. 06

    批处理一周课程

    录音管理面板可批量转写、批量纪要或一键执行两步流程。

Capabilities

专业场景的关键能力，不是 Demo 功能

多 Provider 转写

云端与离线兼容，按场景切换成本、速度、隐私策略。

层级化长文纪要

长课堂先分段摘要再合并，降低上下文长度导致的输出漂移。

播放器内联结果

转写/纪要直接跟随播放器展示，阅读与回听不再跨页面跳转。

可复用 Sidecar 缓存

每个音频对应标准化缓存文件，便于版本管理与二次加工。

批量处理面板

一次处理多个录音，特别适合课程周更、会议周报场景。

中英双语 UI

插件设置内可切换中文/英文界面，适配不同使用习惯。

Quick Start

10 分钟配置完成

  1. 设置页中先配置转写 Provider（Whisper / iFLYTEK / Local）。
  2. 配置纪要 Provider（OpenAI-compatible / Claude）。
  3. 执行 开始/停止录音 获取第一段音频块。
  4. 点击 转写录音 与 生成纪要。
  5. 可选开启 Auto Transcribe 与 Auto Summarize。

Configuration

关键设置建议

录音参数

默认推荐 wav + standard，兼容性与质量更稳。

自动化开关

确认额度和 API 稳定后，再开启自动转写和自动纪要。

长文本策略

保持分段总结开启，90 分钟以上课程建议调低分段阈值。

波形渲染

设备性能偏弱时可关闭波形或降低大小阈值减少卡顿。

FAQ

常见问题

录音失败或提示麦克风权限不足

检查系统麦克风授权与 Obsidian 权限，重启后重试。

转写/纪要提示配置缺失

确认当前所选 Provider 对应的 API Key、URL、模型名已填写完整。

纪要内容为空

先确认转写结果是否包含有效文本；纪要生成依赖转写内容。
